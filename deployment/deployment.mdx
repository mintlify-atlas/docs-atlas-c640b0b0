## Deployment Presets

Nuxt can be deployed to several cloud providers with minimal configuration using deployment presets.

### Configuring Presets

You can explicitly set the desired preset in your `nuxt.config.ts` file:

```ts nuxt.config.ts
export default defineNuxtConfig({
  nitro: {
    preset: 'node-server',
  },
})
```

Alternatively, use the `NITRO_PRESET` environment variable when running `nuxt build`:

```bash
NITRO_PRESET=node-server nuxt build
```

## Rendering Modes

Understanding Nuxt's rendering modes helps you choose the right deployment strategy:

### Server-Side Rendering (SSR)

Pages are rendered on the server for each request. This is the default mode and provides:
- SEO-friendly HTML
- Fast initial page load
- Dynamic content per request
- Full access to server APIs

```ts nuxt.config.ts
export default defineNuxtConfig({
  ssr: true, // default
})
```

### Static Site Generation (SSG)

Pages are pre-rendered at build time:

```bash
npx nuxt generate
```

Benefits:
- Fastest possible page loads
- No server required
- Perfect for CDN distribution
- SEO optimized

### Client-Side Rendering (CSR)

Pages are rendered entirely in the browser:

```ts nuxt.config.ts
export default defineNuxtConfig({
  ssr: false,
})
```

<Warning>
  Client-side rendering loses many SEO benefits. Use `<ClientOnly>` components instead to wrap portions that cannot be server-rendered.
</Warning>

## Hybrid Rendering

Nuxt allows you to mix rendering strategies per route using route rules:

```ts nuxt.config.ts
export default defineNuxtConfig({
  routeRules: {
    // Homepage is pre-rendered at build time
    '/': { prerender: true },
    // Product pages are cached for 1 hour
    '/products/**': { swr: 3600 },
    // Blog uses ISR, regenerates in background
    '/blog/**': { isr: true },
    // Admin dashboard is always server-rendered
    '/admin/**': { ssr: true },
    // API routes use edge runtime
    '/api/**': { cors: true, headers: { 'cache-control': 's-maxage=0' } },
  },
})
```

### Route Rule Options

<ResponseField name="prerender" type="boolean">
  Pre-render at build time for static hosting
</ResponseField>

<ResponseField name="ssr" type="boolean">
  Enable or disable server-side rendering for this route
</ResponseField>

<ResponseField name="swr" type="number | boolean">
  Stale-While-Revalidate caching in seconds
</ResponseField>

<ResponseField name="isr" type="number | boolean">
  Incremental Static Regeneration (regenerates in background)
</ResponseField>

<ResponseField name="cache" type="object">
  Cache configuration with maxAge and other options
</ResponseField>

## Environment Variables

All Nuxt deployment presets respect runtime environment variables:

<ParamField path="NITRO_PORT" type="string" default="3000">
  The port to run the server on (also accepts `PORT`)
</ParamField>

<ParamField path="NITRO_HOST" type="string" default="0.0.0.0">
  The host to bind the server to (also accepts `HOST`)
</ParamField>

<ParamField path="NITRO_PRESET" type="string">
  The deployment preset to use during build
</ParamField>

<ParamField path="NITRO_SSL_CERT" type="string">
  Path to SSL certificate for HTTPS (testing only)
</ParamField>

<ParamField path="NITRO_SSL_KEY" type="string">
  Path to SSL key for HTTPS (testing only)
</ParamField>

## CDN and Proxy Configuration

When deploying behind a CDN or proxy like Cloudflare, configure these settings to avoid issues:

<Steps>
  <Step title="Disable Rocket Loader">
    In Cloudflare: **Speed > Settings > Content Optimization**
    
    Uncheck "Rocket Loader™" to prevent script injection that may cause hydration errors.
  </Step>

  <Step title="Disable Email Obfuscation">
    In Cloudflare: **Security > Settings**
    
    Disable "Email Address Obfuscation" to avoid unwanted HTML modifications.
  </Step>
</Steps>

<Warning>
  CDN settings locations on dashboards may change. These settings are critical to prevent unnecessary re-rendering or hydration errors in production.
</Warning>

## Build Process

Understand the build process for different deployment targets:

<Tabs>
  <Tab title="SSR Deployment">
    ```bash
    # Build for production
    npm run build
    
    # Output directory structure:
    # .output/
    #   server/          # Server bundle
    #   public/          # Static assets
    #   nitro.json       # Nitro config
    ```
  </Tab>

  <Tab title="Static Generation">
    ```bash
    # Generate static site
    npm run generate
    
    # Output directory structure:
    # .output/
    #   public/          # Static HTML + assets
    #     index.html
    #     _payload.json
    #     _nuxt/         # JS/CSS bundles
    ```
  </Tab>

  <Tab title="Edge Deployment">
    ```bash
    # Build for edge runtime
    NITRO_PRESET=cloudflare-pages npm run build
    
    # Output optimized for edge workers
    # .output/
    #   _worker.js       # Edge worker bundle
    #   public/          # Static assets
    ```
  </Tab>
</Tabs>
